到底什么是程序

我们来探寻一下程序是什么。

联系到我们的日常生活，我们使用手机、电脑、电视机，我们去银行会有各种操作终端机，这些都是程序的载体，换句话说，程序是运行在这些机器上的。举个具体例子，我们使用的QQ、微信、浏览器都是一个个程序，它们都是通过某种程序语言开发出来的。在程序的这个定义上讲，它们和我们写的HelloWorld并无差别。

有没有想过，我们现代用的机器内部都是些什么构造？用计算机组成原理（计算机科学的一门必修课）的话讲那就是运算器、控制器、存储器、输入设备、输出设备，出售电脑的厂家也许会说CPU、显卡、主板、内存条、硬盘、电源，一个试图理解程序运行的程序员，首先需要理解CPU，我们来看看CPU是什么。

CPU

看看我们的笔记本电脑，如果你是联想，在键盘左下角也许能看到AMD、Intel等标识，这两个是目前世界上最大的两个CPU制造商，AMD——美国AMD半导体公司， Intel——半导体行业和计算创新领域的全球领先厂商。可见它们都和半导体有关，因为CPU内部是由半导体组成的，而半导体工艺的先进程度决定了CPU的速率。

回到正题，我和chatGPT聊了聊，我问：“如何通俗易懂地讲解CPU是什么？”。他给出了如下回复:

CPU（中央处理器）是计算机或电子设备的“大脑”。它负责执行指令和执行各种任务，以使设备正常工作。可以把CPU想象成办公室的经理，负责接收任务、处理任务并将工作分配给不同的部门。 CPU执行许多操作，例如数学计算、将数据从一个地方移动到另一个地方以及根据某些条件做出决策。它通过遵循一系列指令来完成所有这些操作，这些指令就像完成任务的逐步指南。 当您使用智能手机、笔记本电脑或平板电脑等设备时，会与各种应用程序和软件进行交互。CPU负责运行这些应用程序，并确保它们平稳运行。它处理您的输入，例如点击屏幕或键盘输入，并提供适当的输出，例如在屏幕上显示文本或图像。 总之，CPU是计算机的核心组件，它执行指令和执行任务，使设备功能和效率达到最优。它就像计算机的大脑，管理和协调各种活动。

看了上面的回答，感觉还是很抽象。所以我们从一个专业的计算机角度来分析下。

在计算机科学中，CPU是一个大型的状态机，至于状态机请看什么是状态机。状态机接受一个输入，输入改变计算机的状态，各种状态的组合又变成了输出展现在我们眼前。状态机接受什么样的输入呢？

01010101011111111110100000011010000110001101000000010010010010010111111110010010100

如上便是状态机接受的输入，我们的计算机——手机、电脑等设备本质上成天就和这些“0101”打交道。

至于状态机是由什么构成的，涉及到数字电路的知识，而数字电路是由一个个逻辑门组成的，至于什么是逻辑门请看逻辑门。状态机无非是逻辑门的复杂组合，如果你想领略其中风味，可以阅读数字设计和计算机体系结构。

不同的系统芯片包含不同的CPU，而不同厂商的CPU具体设计也不一样，说到底是状态机的组成不一样。但各大厂商之间遵循一个“协议”，那就是指令集架构(ISA)。目前使用最多，人数最广的是三大架构：Arm, x86 和Riscv。至于什么是ISA，这涉及到汇编语言，可以理解为一种架构和一种汇编语言是一一对应的。汇编语言又是什么？在高级程序语言（C，Java，Python）未发明之前，程序员就通过直接编写汇编语言来和计算机打交道，这比较复杂，但便于理解计算机到底是如何工作的。如果想系统学习汇编语言，可以看看汇编语言——王爽，有C语言基础就能看得懂。

汇编语言

回归正题，什么是汇编语言？看到上面那一串让人头疼的010101了吗？汇编语言就是那串零一的“翻译”，汇编代码与状态机能理解的01字节码一一对应。也就是说，我现在有台机器，如果理解了这台机器对应的汇编代码，我就能通过”写汇编代码“、”将汇编代码转化为01字节码“、”将01字节码传给机器“这三个步骤来与机器交互。汇编语言就是为了方便人类阅读和书写的啊！让我们看个例子：

汇编代码（x86)：

.section .data
 .section .text
 .globl _start
_start:
 movl $1, %eax  
 movl $4, %ebx
 int $0x80

以及对应的01字节码：

xxxxxxxxxx
0000000000000000000000011011100000000000000000000010010111011100000011001101

我们更愿意阅读哪个？答案便不言自明了，于是汇编语言就被发明了出来。

至于汇编代码与01码之间的转化是如何发生的，参见x86 Assembly Guide。其实不用理解是怎么转化的，只要理解汇编语言是特定机器能理解的字节码（对应于ISA）的文本表示就行。

指令集架构（ISA）

我们的手机一般是Arm架构，可以看看我们手机的型号，现在美国高通公司生产的芯片居多，高通是基于Arm架构；再看看我们的电脑，有Intel芯片，有AMD芯片，他们都是x86架构。为什么要有这样一个协议呢？因为通过上面的讨论，我们知道只要指令集架构（ISA，协议）相同，同样的汇编代码就能运行在不同的CPU芯片上。既然Intel芯片和AMD芯片都是x86, 那么x86代码运行能同时在这两种芯片上运行也就不足为奇了。这便是一种抽象。通过添加一层指令集架构抽象，我们得到了汇编代码运行的便利性。

这种便利性具体体现在哪呢？假设QQ在一台Intel芯片的电脑上运行的很好，我们把他移动到另一台AMD芯片的电脑上，理想情况下也能运行的很好，这也就免去了软件开发者需要适配不同机器带来的繁琐。

C语言程序

好的，当我们理解了CPU，汇编语言，指令集架构后，在来看看最简单的C语言HelloWorld程序是如何在计算机上运行的吧。以下是HelloWorld程序：

#include <stdio.h>
int main () {
    printf("Hello World\n");
    return 0;
}

或许你已经习惯了集成开发环境（IDE），CLion、VisualStudio、Dev C++、CodeBlock等软件都是适用于C的IDE。他们最大的特点是可通过点击一个按钮来一键执行代码。简单是简单，但并不助于我们理解C语言程序是如何跑起来的。

通过上面的讨论，计算机并不能理解C语言（计算机是一个状态机，状态机只能理解01字节码），那么，C语言又是如何运行在计算机上的呢？

实际上，C语言代码首先通过某个被称为编译器（compiler)的程序（gcc, clang, msvc) 等转化为汇编代码。例如gcc这个软件就有不同版本，为什么呢？因为在不同的机器上，我们需要将C语言转化为不同的汇编代码。为什么需要将C语言转化为不同的汇编代码呢？因为一种机器只能理解一种汇编代码。

好的，我们得到了汇编代码。不得不说另一个工具了，那就是汇编器（assembler)。汇编器将汇编代码转化为机器码也就是上面说的01码，这样机器就能理解了。

我们的HelloWorld程序经过这样一系列转化，最终变成了一串01码，这样计算机终于能理解了，我们也因此能通过双击main.exe文件来运行这个程序了。

抽象

计算机系统无处不是抽象。

在上面的讨论中，我们发现ISA的意义就在于：可以使同一个汇编代码运行在不同的CPU上，只要CPU都实现了这个指令集。于是我们看到，一个x86架构的软件（汇编代码是x86）既可以运行在Intel电脑，也可以运行在AMD电脑上，甚至可以运行在x86虚拟机上以提供便利。这就是说，ISA以及对应的汇编语言在CPU上建立了一层抽象，使得CPU设计者少操心了许多兼容问题，只要保证CPU实现的是某种ISA，就能保证能让相应程序正确运行。

我们也提到了gcc，有各种不同的gcc，负责将C代码翻译成各种ISA下的机器码（字节码），这意味着什么呢？这意味着，我写了一个C语言程序跑在了x86系统上，我不用改动（或者做一些小的改动）就可以运行在arm、riscv等架构的其他系统上，这大大减少了软件移植的时间。这一切都是gcc这个软件完成的：一旦未来某种架构进行了更新或是新出了一种架构，我只要重写gcc的代码，就能实现程序的高效移植。于是，我们可以说，gcc（编译器）帮助高级语言在不同系统上建立了一层抽象，使得程序员不用考虑宿主系统的ISA，也能编写出可执行的程序。

总结

HelloWorld是如何执行的我们知道了，同理也就能理解其他任何程序的执行了。任何高级语言编写的程序，都需要经过一系列过程，转化为特定机器上的字节码，才能被执行。我们接触的任何程序（也就是软件）、游戏，都是这么被运行的，放大了看，进入计算机内部看，他们无非是一串串0101字节码，无非是一波波高低的电信号（0101在计算机内部是通过电平高低表示的）。这些电信号改变了计算机内部的状态，而这些状态又与即将进入的一波波信号交互，达到新的状态，这些状态或可见，或不可见，可见的部分展现在了我们的眼前，而大量的不可见部分则是计算机内部得以恒久运行的基础。
